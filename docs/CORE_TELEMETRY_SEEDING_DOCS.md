# Core Telemetry Seeding

## Purpose

This document describes the reduced seeding model implemented by `generate_core_synthetic_telemetry.py`.

This version intentionally focuses on a small set of raw locomotive telemetry signals:

- speed
- adhesion coefficient
- fuel rate and fuel level
- key pressures
- key temperatures
- voltage and current
- raw faults

It does not generate:

- GPS
- heading
- acceleration as an output column
- battery voltage
- communications telemetry
- health index
- derived alerts tables
- dispatcher command tables

## Output

The script writes one telemetry dataset to:

- `synthetic_output_core/telemetry.csv`
- `synthetic_output_core/telemetry.jsonl`
- optional `synthetic_output_core/telemetry.parquet`

## Locomotives

The seed set contains exactly three locomotives:

1. `KZ8A-001`
2. `TE33A-002`
3. `KZ8A-003`

This guarantees at least one electric and one diesel-electric locomotive.

## Determinism

- Global seed: `RANDOM_SEED = 42`
- Per-locomotive seed: `RANDOM_SEED + index * 10_000`

Repeated runs with the same code and dependency versions will produce the same telemetry.

## Hidden State Model

The generator uses a compact hidden-state model. The hidden state is not written to the dataset, but it drives all output columns.

### Operating phases

Each locomotive cycles through:

1. `idle`
2. `accel`
3. `cruise`
4. `brake`
5. `stop`

### Hidden drivers

The generated telemetry is driven by:

- speed target
- positive acceleration
- braking demand
- load signal
- thermal inertia

## Connection Rules Between Parameters

The output columns are not independent random values.

### 1. Speed

- `speed_kmh` is generated by phase and then smoothed
- `accel` raises speed gradually
- `cruise` keeps speed in a band with small oscillation
- `brake` reduces speed gradually
- `stop` and `idle` keep speed near zero

### 2. Traction current

- `traction_current_a` is low near stop
- it rises sharply during acceleration
- it remains moderate during cruise
- it drops during braking
- sustained high current contributes to higher temperatures
- lower `adhesion_coeff` reduces effective traction current under the same demand

### 3. Brake pressures

- `brake_cylinder_pressure_bar` rises when braking demand rises
- `brake_pipe_pressure_bar` drops as brake cylinder pressure rises
- braking pressure changes must align with falling speed
- weak adhesion can reduce braking effectiveness and keep brake pipe pressure relatively higher than expected

### 4. Adhesion

- `adhesion_coeff` represents rail-wheel adhesion quality
- it changes slowly, not as white noise
- it is usually in a healthy band and occasionally dips for a sustained window
- lower adhesion weakens effective traction response
- lower adhesion also weakens braking response at speed

### 5. Temperatures

- `traction_motor_temp_c` responds relatively quickly to sustained traction load
- `bearing_temp_c` changes more slowly and with lower amplitude
- for `KZ8A`, `transformer_temp_c` follows sustained electrical load
- for `TE33A`, `coolant_temp_c` follows sustained diesel load
- temperatures cool gradually when load falls

### 6. KZ8A electrical rules

- `catenary_voltage_kv` is mostly stable with moderate noise
- higher load causes mild voltage sag
- high sustained load raises `transformer_temp_c`

### 7. TE33A fuel and pressure rules

- `fuel_rate_lph` rises with load
- `fuel_level_l` is the cumulative inverse of fuel burn and only decreases
- `oil_pressure_bar` stays in a normal band under healthy operation
- thermal or anomaly stress can lower oil pressure

### 8. Fault rules

- `fault_code` is null most of the time
- it becomes non-null only during anomaly windows
- faults are sustained windows, not single samples

## Injected Anomalies

The generator modifies raw telemetry directly during anomaly windows.

### Common anomaly

- `BRAKE_RESPONSE_WEAK`

Effects:

- brake cylinder pressure is weaker than expected during a braking window
- brake pipe pressure stays too high
- this creates a realistic basis for a high-speed braking alarm

### KZ8A anomalies

- `KZ8A_VOLTAGE_DIP`
- `KZ8A_TRANSFORMER_HOT`

Effects:

- catenary voltage drops below normal
- transformer temperature rises above normal trend

### TE33A anomalies

- `TE33A_OIL_LOW`
- `TE33A_COOLANT_HOT`

Effects:

- oil pressure drops below normal
- coolant temperature rises above normal
- fuel burn can rise during stressed operation

## Schema

| Column | Type | Nullable | Applies To | Description |
|---|---|---:|---|---|
| `timestamp` | `TIMESTAMP(3)` or text ISO 8601 | No | all | Event time with millisecond precision |
| `t_ms` | `BIGINT` | No | all | Relative time in milliseconds from stream start |
| `locomotive_id` | `VARCHAR(32)` | No | all | Stable locomotive ID |
| `locomotive_type` | `VARCHAR(16)` or enum | No | all | `KZ8A` or `TE33A` |
| `speed_kmh` | `REAL` | No | all | Train speed |
| `adhesion_coeff` | `REAL` | No | all | Effective wheel-rail adhesion coefficient |
| `traction_current_a` | `REAL` | No | all | Traction current |
| `brake_pipe_pressure_bar` | `REAL` | No | all | Brake pipe pressure |
| `brake_cylinder_pressure_bar` | `REAL` | No | all | Brake cylinder pressure |
| `traction_motor_temp_c` | `REAL` | No | all | Traction motor temperature |
| `bearing_temp_c` | `REAL` | No | all | Bearing temperature |
| `fault_code` | `VARCHAR(64)` | Yes | all | Raw fault code |
| `catenary_voltage_kv` | `REAL` | Yes | KZ8A | Overhead supply voltage |
| `transformer_temp_c` | `REAL` | Yes | KZ8A | Transformer temperature |
| `fuel_level_l` | `REAL` | Yes | TE33A | Remaining fuel volume |
| `fuel_rate_lph` | `REAL` | Yes | TE33A | Fuel burn rate |
| `oil_pressure_bar` | `REAL` | Yes | TE33A | Engine oil pressure |
| `coolant_temp_c` | `REAL` | Yes | TE33A | Coolant temperature |

## Null Handling

Type-specific numeric fields are generated as `np.nan` when they do not apply.

Meaning:

- JSONL output writes them as `null`
- CSV output leaves them empty
- they represent "not applicable", not zero

## Suggested Raw Table

```sql
create table telemetry_raw_core (
    timestamp timestamptz not null,
    t_ms bigint not null,
    locomotive_id varchar(32) not null,
    locomotive_type varchar(16) not null,
    speed_kmh real not null,
    adhesion_coeff real not null,
    traction_current_a real not null,
    brake_pipe_pressure_bar real not null,
    brake_cylinder_pressure_bar real not null,
    traction_motor_temp_c real not null,
    bearing_temp_c real not null,
    fault_code varchar(64) null,
    catenary_voltage_kv real null,
    transformer_temp_c real null,
    fuel_level_l real null,
    fuel_rate_lph real null,
    oil_pressure_bar real null,
    coolant_temp_c real null,
    primary key (locomotive_id, timestamp)
);
```
